# Features (Human Description)

## Реализованные функции (v2.0)

### Управление плейлистами
- ✅ **Создание плейлистов**: `/create_playlist <название>` — создание нового плейлиста в Яндекс.Музыке
- ✅ **Мои плейлисты**: `/my_playlists` — просмотр плейлистов, созданных пользователем
- ✅ **Общие плейлисты**: `/shared_playlists` — просмотр плейлистов, куда пользователь добавляет треки
- ✅ **Информация о плейлисте**: `/playlist_info [номер]` — детальная информация о плейлисте
- ✅ **Редактирование названия**: `/edit_name <новое название>` — изменение названия (только для создателя)
- ✅ **Удаление плейлиста**: `/delete_playlist` — удаление из базы данных бота (только для создателя)

### Работа с треками
- ✅ **Добавление треков**: Отправка ссылки на трек, альбом или плейлист автоматически добавляет их в активный плейлист
- ✅ **Просмотр треков**: `/list [номер плейлиста]` — просмотр списка треков в плейлисте
- ✅ **Удаление треков**: `/queen_liza <номер>` — удаление трека по номеру из активного плейлиста

### Шаринг и доступ
- ✅ **Шаринг плейлистов**: При создании плейлиста генерируется уникальная ссылка для шаринга
- ✅ **Получение доступа**: Переход по ссылке или отправка токена боту предоставляет доступ на добавление треков
- ✅ **Управление доступом**: Создатель имеет полные права, участники могут только добавлять треки

### Персонализация
- ✅ **Свой токен**: `/set_token <токен>` — использование собственного аккаунта Яндекс.Музыки для создания плейлистов
- ✅ **Активный плейлист**: Автоматическое определение активного плейлиста для добавления треков

### База данных
- ✅ **SQLite**: Локальная база данных для хранения пользователей, плейлистов, доступов и истории действий
- ✅ **Автоматическая инициализация**: База данных создается автоматически при первом запуске

### Интерфейс
- ✅ **Клавиатура**: Reply-клавиатура с основными командами
- ✅ **Inline-кнопки**: Выбор плейлиста через интерактивные кнопки
- ✅ **Контекст пользователя**: Сохранение активного плейлиста для каждого пользователя

## Планируемые функции

- ⏳ **FSM-логика**: Интерактивные диалоги для создания плейлистов и других действий
- ⏳ **Установка обложки**: Загрузка обложки для плейлиста через фото
- ⏳ **Рекомендации**: Получение рекомендаций на основе плейлиста
- ⏳ **PostgreSQL**: Миграция на PostgreSQL для продакшена
- ⏳ **Docker**: Docker-окружение для быстрого разворачивания
- ⏳ **Улучшенный UX**: Более удобные диалоги и обработка ошибок
